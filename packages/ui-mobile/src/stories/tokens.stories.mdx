Design tokens are the visual design atoms of the design system â€” specifically, they are named entities that store visual design attributes. We use them in place of hard-coded values (such as hex values for color or pixel values for spacing) in order to maintain a scalable and consistent visual system.

## Usage Guidelines

### Color Usage
- Use primary colors for main actions and brand elements
- Use secondary colors for supporting elements
- Use semantic colors (success, warning, error) for status indicators
- Ensure proper contrast ratios for accessibility (4.5:1 for normal text, 3:1 for large text)

### Typography Usage
- Use consistent font sizes from the scale
- Maintain proper line height for readability
- Use font weights to establish hierarchy

### Spacing Usage
- Use the 4px grid system for consistent spacing
- Apply spacing tokens instead of arbitrary values
- Maintain consistent margins and padding

### Border Radius Usage
- Use consistent border radius values
- Match border radius to component size and context
- Maintain visual consistency across similar components
